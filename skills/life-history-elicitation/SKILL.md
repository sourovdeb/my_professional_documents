# Life History Elicitation

## Purpose
Protocol for conducting rigorous, ethically sound life-history interviews that treat recall as reconstruction (Bartlett 1932; Schacter's seven sins of memory). Vividness ≠ accuracy (Neisser & Harsch 1992). Guards against interviewer suggestion (Loftus misinformation effect), imagination inflation, hindsight bias, childhood amnesia boundary (~age 3), reminiscence bump, confidence–accuracy gap, and narrative smoothing.

Output is a **labeled ledger** (RECOLLECTION / LORE / INFERRED), not a tidied story. Tidying is a visible downstream writing decision. Facts closed before meaning questions.

## When to Use
Activate on test prompts like:
- "Interview me about my Sydney years for the autoethnography"
- Any request for biographical/life-story elicitation in context of autoethnographic research, disability/education case studies, or personal narrative for regulatory/appeal documentation.
Chains ahead of writer skills, socratic-self-review, and publish steps. Fits "Disclosure Without Demand".

## Method Stack (One line each — all named protocols)

| Step | Protocol | Why it beats naive Q&A |
|------|----------|------------------------|
| Landmark calendar first | Life History Calendar (Freedman 1988; Belli 1998) | Dates anchored to residences/jobs/events — kills telescoping |
| Chapters → free narrative | McAdams Life Story Interview; Cognitive Interview "report everything" | Uninterrupted recall before any probe; fragments kept |
| Context reinstatement | Cognitive Interview (Fisher & Geiselman 1992) | Reinstating place/people/feeling measurably raises correct recall |
| Funnel probing | PEACE (Account phase) | Open → cued → closed last; forced-choice banned early |
| Reverse-order pass | Cognitive Interview | Different retrieval route surfaces missed detail; perspective-change technique excluded (confabulation risk) |
| Source-monitoring pass | Johnson, Hashtroudi & Lindsay 1993 | "Remember it, told it, or pieced it together?" → RECOLLECTION / LORE / INFERRED |
| Consistency read-back | Investigative-interview practice | Contradictions flagged neutrally; "I don't know" accepted — pressure manufactures false memories |

Directly encodes guards for: suggestibility (Loftus), imagination inflation, hindsight bias, childhood-amnesia boundary, reminiscence bump, confidence–accuracy gap, narrative smoothing.

## Core Guardrails (Ethics & Accuracy)
- **No writer/interviewer skill visible** among loaded skills — this skill chains ahead of any writer/ledger → writer → socratic-self-review → publish.
- Protocol citations are RECOLLECTION-ONLY. Verify before quoting in papers (no web egress on restricted sessions).
- Persist `.skill` + `.md` to repo/Drive — session outputs expire.
- Never pressure for detail. Accept "I don't know". Neutral flagging only.
- Separate facts from meaning/interpretation questions.
- Output format: Labeled ledger with source tags, not polished narrative.

## Instructions for Activation
1. Confirm user context matches autoethnography / personal case study / regulatory narrative.
2. Begin with Life History Calendar anchoring (residences, jobs, key events with dates).
3. Proceed through free narrative chapters.
4. Apply context reinstatement, funnel, reverse-order, source-monitoring passes.
5. Label every statement: RECOLLECTION (I remember experiencing it), LORE (told to me), INFERRED (pieced together).
6. End with neutral consistency read-back.
7. Deliver raw ledger to user or downstream writer skill.
8. Combine with autoethnographic-research and authority-letters skills for CELTA/disability case documentation.

Do not modify core protocols. Adapt only surface parameters (period, names, locations) to context.
