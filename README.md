# my_professional_documents — Sourov Deb

The single home for my professional life: my story, my evidence, my tools, and the
engine that turns writing into exposure, contacts, and income. Built to run on my
worst day, not just my best — every system here assumes uneven energy (bipolar I,
ADHD, insomnia) and carries the day when I can't.

**Quality over quantity. Active voice. One real thing a day.**

## Start here

1. Open **[`Growth_Hub/00_START_HERE.md`](Growth_Hub/00_START_HERE.md)** — the writing,
   exposure, tools, ideas, contacts, consistency, wellbeing, and automation system.
2. Write the 500 words. Publish. Log it in
   [`Growth_Hub/PUBLISHED_LOG.md`](Growth_Hub/PUBLISHED_LOG.md).
3. Let the automation surface jobs and like-minded people while I write.
4. On a crash day, follow the four anchors in the Daily Living Strategy. That still counts.

## Conventions going forward

- **One home per thing.** New writing → `posts/drafts/` (or `Growth_Hub/drafts/`).
  New ideas → `Growth_Hub/IDEA_INBOX.md`.
- **Plain `.md`** for anything I'll edit or want diffed in git.
- **Decide the structure once, on a stable day, then stop redesigning it.**
- **No secrets in git.** Credentials (WordPress/FTP, API keys, SMTP) go in environment
  variables or a local untracked file — never committed. There are none in this repo today.
- **One branch per creation** when drafting publicly, so I review before anything ships.
