# WordPress Health — Monitoring Guide

This document explains how to keep your WordPress site healthy, what to monitor,
and how to interpret the automated health reports generated by `scripts/wp_health_check.py`.

---

## What the Health Check Tests

| Check | What it verifies | Why it matters |
|-------|-----------------|----------------|
| **WP REST API** | `/wp-json/` returns valid JSON | Core of all automation |
| **Custom endpoint** | `/wp-json/sourov/v1/health` responds | Your plugin is active |
| **Home page** | Returns HTTP 200 with content | Site is reachable |
| **SSL certificate** | HTTPS valid, not expired | Security + SEO ranking |
| **robots.txt** | File exists and has Disallow directives | Search engine guidance |
| **Sitemap** | XML sitemap found | Search engine indexing |
| **RSS feed** | `/feed/` returns valid XML | Syndication / IFTTT |
| **Login page** | `wp-login.php` is accessible | Access control audit |

---

## Running the Health Check

```bash
# One-time check
python scripts/wp_health_check.py

# From Makefile
make health

# From cron (weekly on Sunday at 6 AM)
0 6 * * 0 /usr/bin/python3 /path/to/scripts/wp_health_check.py
```

Output saved to: `docs/health_report_YYYY-MM-DD.json`

## Reading the Report

```json
{
  "timestamp": "2026-06-03T08:00:00Z",
  "site": "https://yourdomain.com",
  "score": "87% (7/8)",
  "checks": [
    { "check": "WP REST API", "status": "pass", "ms": 342 },
    { "check": "Sitemap",     "status": "fail", "detail": "No sitemap found" }
  ]
}
```

- **score** — percentage of checks that passed
- **ms** — response time in milliseconds (>2000ms = slow)
- **status: fail** — read the `detail` field for the specific error

## Common Issues and Fixes

### REST API returns 401 / 403
- Check that your API key is correct in `scripts/.env`
- Verify the `sourov-ai-controller.php` plugin is active in WordPress admin
- Some security plugins (Wordfence, iThemes) block REST API — whitelist your IP

### No sitemap found
- Install **Yoast SEO** or **Rank Math** — both generate sitemaps automatically
- Or enable Gutenberg sitemap: Settings > Reading > Enable sitemap

### SSL check fails
- SSL certificate may be expired — renew via Let's Encrypt: `certbot renew`
- Or contact your hosting provider (Hostinger usually auto-renews)

### Home page is slow (>3000ms)
- Install **WP Super Cache** or **W3 Total Cache** plugin
- Enable Gzip compression in `.htaccess`
- Optimise images with **Squoosh** or **ShortPixel** plugin

### Login page check shows unexpected status
- HTTP 200 = normal (page is accessible)
- HTTP 301/302 = redirect (check where it goes)
- HTTP 404 = login page moved (some security plugins rename it) — this is fine

## WordPress Maintenance Checklist (Monthly)

- [ ] Run `make health` and review the report
- [ ] Update WordPress core (`wp core update`)
- [ ] Update all plugins (`wp plugin update --all`)
- [ ] Update theme (`wp theme update --all`)
- [ ] Check database for errors (`wp db check`)
- [ ] Verify backup is working (UpdraftPlus or rclone)
- [ ] Review error log: `wp eval 'print_r(wp_debug_log());'`
- [ ] Check disk space on hosting server
- [ ] Test post creation via REST API (use `testConnection()` in Apps Script)
- [ ] Review Google Search Console for crawl errors

## WP-CLI Maintenance Commands

```bash
# Update everything
wp core update && wp plugin update --all && wp theme update --all

# Optimise database
wp db optimize

# Clean up revisions, drafts, spam
wp post delete $(wp post list --post_status=trash --format=ids)
wp comment delete $(wp comment list --status=spam --format=ids)

# Flush cache
wp cache flush
wp rewrite flush

# Check for broken links in posts
wp eval 'foreach(get_posts(["numberposts"=>-1]) as $p) echo $p->guid."\n";'

# View recent error log entries
tail -n 50 /wp-content/debug.log
```

## Security Hardening

```bash
# Change default admin username
wp user update 1 --user_login=new_admin_name

# Disable file editing from WP admin
wp config set DISALLOW_FILE_EDIT true --type=constant

# Force HTTPS
wp config set FORCE_SSL_ADMIN true --type=constant

# List users with admin role
wp user list --role=administrator
```

**Recommended security plugins:**
- **Wordfence** — firewall + malware scanner (free)
- **WP Hide Login** — rename `wp-login.php`
- **Limit Login Attempts Reloaded** — brute-force protection

2. Add Monitor → HTTP(s) → paste your site URL
3. Add alert contact → your email
