# SKILL: CELTA-Level Teacher (Lesson Design Across Contexts)
**Version:** 1.0 | **Created:** 2026-06-14 | **Author:** Sourov DEB
**Scope:** Plan and teach English lessons to CELTA methodology standard, adapted to the learner category and environment (young learners → business → exam → EAP → ESOL → one-to-one). The general lesson engine; pairs with SKILL_elt_essay_teacher (writing) and SKILL_elt_elicitor (questioning).
**Authority note:** Encodes the Cambridge CELTA syllabus and the ELT methodology canon cited under SOURCES — not the author's credential. Per project record, the author's own CELTA is **"cours complété, 120 heures supervisées, 4 travaux écrits validés au standard requis, qualification en appel"** — never "awarded/certified." This skill teaches the *method*, which is public.

---

## OVERVIEW
A CELTA-standard lesson is judged on a fixed set of things: clear staged aims, sound language analysis (Meaning–Form–Pronunciation), learner-centredness (learners talk more than the teacher), anticipated problems with solutions, and fit to level and context. This skill produces that plan and adapts every variable to *who* you are teaching.

---

## TRIGGER CONDITIONS
Activate when the user wants to:
- Plan an English lesson or a scheme of work.
- Teach a specific grammar point, lexical set, function, or skill (reading/listening/speaking/writing).
- Adapt a lesson for a particular learner type (kids, teens, adults, business, exam, academic, migrants, 1-to-1, large or mixed-ability classes).
- Self-check a plan against CELTA assessment expectations.

## MODULE 1 — THE CELTA FRAME (what every plan must show)
```
CELTA syllabus = 5 topics (Cambridge, 5th ed. 2018):
  T1 Learners/teachers & the learning context   T2 Language analysis & awareness
  T3 Language skills (R/L/S/W)                   T4 Planning & resources for contexts
  T5 Developing teaching skills & professionalism
A sound plan demonstrates:
  - main aim + subsidiary aims (what learners will be able to DO by the end)
  - a staged procedure: each stage has a purpose, timing, and interaction pattern
  - language analysis (Module 4) for the target language
  - anticipated problems + solutions (Module 7)
  - learner-centred design: STT > TTT; clear, checked instructions; context before form
```

## MODULE 2 — LEVEL PLACEMENT (grade everything to CEFR)
```
A1 A2  Basic User        B1 B2  Independent User       C1 C2  Proficient User
Grade language (your own talk), task demand, text difficulty, and success criteria to level.
Same topic, different lesson at A2 vs C1. Mismatch = the commonest planning fault.
Source: Council of Europe CEFR (2001) + Companion Volume (2020).
```

## MODULE 3 — FRAMEWORK SELECTION (pick by aim + learner)
```
PPP   Presentation → Practice → Production      new, discrete language; lower levels; clear target
ESA   Engage → Study → Activate (Harmer)        flexible; can re-order (e.g. boomerang/patchwork)
TTT   Test → Teach → Test                       when learners may already half-know the item
TBL   task → report → language focus (Willis)   fluency, communication, stronger/older learners
GUIDED DISCOVERY  examples → learners infer rule → confirm   grammar, analytical/higher levels
RECEPTIVE SKILLS  lead-in → gist → detail → response/follow-on   reading & listening lessons
PWP   Pre → While → Post                        skills frame (pre-teach, set tasks, exploit)
Rule: framework serves the aim. Don't force PPP onto a fluency or a reading lesson.
```

## MODULE 4 — LANGUAGE ANALYSIS: MFP (CELTA Topic 2)
```
For every target item, analyse:
  MEANING        what it means + HOW you'll convey and CHECK it (context + CCQs — see
                 SKILL_elt_elicitor). Anticipate confusion with related items.
  FORM           structure (positive/negative/question), part of speech, written form,
                 and the SPOKEN form (contractions, weak forms).
  PRONUNCIATION  word/sentence stress, connected speech, problem sounds. Plan to drill it.
No item goes into a lesson without MFP done first. This is what trainees most often skip.
```

## MODULE 5 — LEARNER-CATEGORY PLAYBOOK (the "different environments/students")
```
CATEGORY            WHAT CHANGES (aims · staging · materials · interaction)
─────────────────────────────────────────────────────────────────────────────────────
YOUNG LEARNERS      short stages, routines, play, songs, TPR (physical response); visual/
(approx 5–11)       concrete; lots of repetition; manage attention not grammar metalanguage.
                    (Asher TPR; Cameron, Teaching Languages to Young Learners.)
TEENS               relevance + autonomy; topics they own; manage group dynamics; clear
                    purpose or they disengage; balance control and freedom.
ADULTS — GENERAL    communicative, life-relevant tasks; respect prior knowledge; clear "why".
                    (CLT — communicative language teaching.)
BUSINESS / ESP      NEEDS ANALYSIS FIRST (Hutchinson & Waters): teach their actual tasks
                    (emails, meetings, calls, presentations); authentic materials; ROI framing;
                    register over volume.
EXAM CLASSES        teach the rubric + exam strategy + timed practice (positive backwash);
                    diagnose against the band/grade descriptors; little-and-often timed work.
EAP                 academic register, genre, citation, argument, listening to lectures;
                    long-term writing/reading focus.
ESOL / MIGRANTS     survival + functional English; literacy-sensitive (some learners
                    low-literate in L1); life-admin contexts; pace and dignity.
ONE-TO-ONE          negotiated syllabus; maximal personalisation; learner sets agenda;
                    no peer interaction — you ARE the interlocutor; more feedback, less STT loss.
LARGE / MULTILING   pair/group work to raise STT; strong monitoring; mingles; instructions
                    must be watertight (ICQ-checked); use the shared L1 absence as an asset.
MIXED-ABILITY       differentiated tasks; tiered questions; fast-finisher extensions;
                    group strong+weak strategically.
```
For each cell, the *frame* (Module 3) and *level* (Module 2) still apply — this table only sets what shifts.

## MODULE 6 — STAGING & INTERACTION
```
- LEAD-IN first: generate interest + context before any target language.
- Each stage: purpose · timing · interaction pattern (T-S / S-S / pair / group / open class).
- STT MAXIMISATION: aim learners talking ~70–90% of speaking time; teacher ~10–30%.
- INSTRUCTIONS: short, sequenced, demonstrated; check with ICQs (SKILL_elt_elicitor), not "OK?".
- MONITORING: circulate, note errors for delayed correction, support without taking over.
- FEEDBACK stage on content AND language (delayed error correction from monitoring notes).
```

## MODULE 7 — ANTICIPATED PROBLEMS + SOLUTIONS (CELTA-assessed)
```
For the lesson, predict and pre-solve:
  LINGUISTIC  meaning confusions, form errors, pronunciation traps (from Module 4).
  SKILLS      task too hard/easy; unfamiliar text topic; timing overrun.
  LOGISTICAL  unclear instructions; uneven pairs; tech failure; early finishers.
Write the solution next to each. "Anticipated problems with solutions" is an explicit
CELTA criterion and the difference between a plan and a wish.
```

## OUTPUT STRUCTURE (a complete lesson plan)
```
1. CONTEXT: level (CEFR), learner category, class size, lesson length, lesson type.
2. AIMS: main aim (learners will be able to…) + subsidiary aims.
3. TARGET LANGUAGE + MFP analysis (Module 4), with the CCQ set (from SKILL_elt_elicitor).
4. ANTICIPATED PROBLEMS + SOLUTIONS (Module 7).
5. PROCEDURE TABLE: stage | aim | time | interaction | teacher does | learners do.
6. MATERIALS list (+ any board plan / handout).
7. CATEGORY ADAPTATION notes (what this plan changes for the learner type — Module 5).
```

## FAILURE MODES
- Level mismatch (same lesson at A2 and C1) → grade everything to CEFR (Module 2).
- No MFP → meaning/form/pron problems surface live, unplanned (Module 4).
- Teacher-centred (high TTT) → redesign stages for pair/group work; cut teacher talk.
- No anticipated problems → the plan is a wish; add Module 7.
- Forcing one framework onto every aim → pick by aim + learner (Module 3).
- Skipping needs analysis for business/ESP → you teach the wrong English (Module 5).
- Describing the author's own CELTA as "certified/awarded" anywhere → use the appeal-status phrasing.
