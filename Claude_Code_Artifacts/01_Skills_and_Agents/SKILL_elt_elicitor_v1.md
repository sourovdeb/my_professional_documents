# SKILL: ELT Elicitor (Elicitation & Understanding Checks)
**Version:** 1.0 | **Created:** 2026-06-14 | **Author:** Sourov DEB
**Scope:** The questioning toolkit — draw language/knowledge from learners instead of telling (elicitation), check meaning (CCQs), check instructions (ICQs), run guided discovery, drill form. The single most CELTA-load-bearing classroom technique. Sub-skill of SKILL_celta_lesson_teacher.
**Why this is its own skill:** It maps directly to CELTA criterion 5g — *"using a range of questions effectively for the purpose of elicitation and checking of understanding"* — and it is the technique trainees most often get wrong.

---

## OVERVIEW
Three question jobs are constantly confused. **Eliciting** questions surface what learners *already* know. **CCQs** check understanding of *newly presented* language. **ICQs** check learners understood the *task*. "Do you understand?" checks nothing. This skill keeps them separate and produces correct sets for a given language point.

---

## TRIGGER CONDITIONS
Activate when the user wants to:
- Elicit language or ideas from learners rather than explain.
- Write concept-checking questions (CCQs) for a grammar point or vocabulary item.
- Write instruction-checking questions (ICQs) for a task.
- Run guided discovery, drill pronunciation/form, or raise student talking time (STT).
- Diagnose why learners "nod yes" but haven't understood.

---

## MODULE 1 — ELICIT BEFORE YOU TELL (the principle)
```
WHY: raises STT, activates prior knowledge, keeps learners cognitively active, and tells you
     what they already know so you don't teach what they have.
DISPLAY question  = teacher knows the answer, learner knows teacher knows (most classroom Qs).
REFERENTIAL question = genuine, teacher doesn't know (raises real communication).
Use a strong context/visual/situation so the language is RETRIEVABLE; you can't elicit what a
learner has never met (Module 7 anti-pattern). Capture elicited forms on the board.
(British Council; Scrivener.)
```

## MODULE 2 — THREE QUESTION TYPES (keep them separate)
```
                 PURPOSE                                  EXAMPLE
ELICITING Q   surface what learners ALREADY know      [show a banana] "What's this?" → banana
CCQ           check meaning of NEW language just       target "He's been to Japan":
              presented (NOT to make them say the TL)    Is he in Japan now? (No)
                                                          Did he go at some time before now? (Yes)
                                                          Do we know exactly when? (No)
ICQ           check they understood the TASK            "Are you writing or speaking?"
              instructions                               "Alone or in pairs?"  "How many minutes?"
DO NOT ASK    "Do you understand?" / "OK?" — a "yes" is unreliable; it can mean
              'I think so', 'I don't want to lose face', or 'I stopped listening'.
              (British Council, Cooley.)
```

## MODULE 3 — HOW TO ELICIT WELL
```
1. Build the concept first (context, picture, situation, story) so retrieval is possible.
2. Ask a focused prompt aimed at one thing.
3. If elicitation succeeds → confirm, board it, move on.
4. If it FAILS → narrow the prompt once; if still nothing, SUPPLY the item and re-check with
   a CCQ. Do not labour a dead elicitation (wastes time, lowers energy).
5. Personalise where possible (referential questions) to make it real, not a guessing game.
6. Nominate across the room — strong and weak learners — not only volunteers.
```

## MODULE 4 — WRITING CCQs (procedure + rules)
```
RULES (Thornbury, An A–Z of ELT; British Council):
  - Use SIMPLE language (simpler than the target).
  - DO NOT use the target language in the CCQ.
  - Aim for yes/no or one-word answers.
  - Concrete nouns usually need NO CCQ — use a picture instead.
  - Ask 3–4 CCQs, not one. One right answer from one learner ≠ class understanding.
  - Aim some CCQs at weaker learners.

PROCEDURE:
  1. Break the target MEANING into its concept components.
  2. Write one yes/no question per component.
  3. Strip out the target language; replace with plain words.
  4. Test against a DISTRACTOR meaning: if a CCQ is also "yes" for the wrong meaning, it's broken.

WORKED PATTERNS:
  Past simple "I visited Rome last year":
     In the past? (Yes) · Am I there now? (No) · Do we know when? (Yes, last year)
  Present perfect (experience) "She has eaten snails":
     Now or before? (Before) · Do we know when exactly? (No) · Once is enough? (Yes)
  2nd conditional "If I had time, I'd travel":
     Do I have time? (No) · Real or imaginary? (Imaginary) · Present or past meaning? (Present)
  Modal obligation "You must wear a seatbelt":
     Is it a choice? (No) · Necessary? (Yes) · Who says so — a rule or a wish? (A rule)
  Vocabulary connotation "stubborn":
     Is it positive or negative? (Negative) · Does the person change their mind easily? (No)
  CULTURALLY-LOADED / no L1 equivalent (e.g. perfect aspect, "subway" UK vs US): CCQs are
     most valuable here — the concept differs or doesn't exist in the L1.
```

## MODULE 5 — GUIDED DISCOVERY
```
Give examples or a short text containing the target → ask GRADED questions so learners infer
meaning + form themselves → confirm and refine. Best for grammar with analytical / higher-level
learners (inductive teaching). Pairs with CCQs to verify the inferred concept.
```

## MODULE 6 — DRILLING (CCQs check meaning, NOT pronunciation — drill it)
```
CHORAL first (safety in numbers) → INDIVIDUAL (check it stuck).
BACKCHAINING for connected speech: ...museum / to the museum / go to the museum.
SUBSTITUTION drills to practise form. Model clearly; mark word/sentence STRESS and weak forms.
Drill before freer practice so production isn't blocked by form/phonology.
```

## MODULE 7 — ANTI-PATTERNS (the trainee failure list)
```
- Eliciting the impossible: asking learners to define/produce language they've never met.
- CCQs that contain the target language (self-defeating — they reveal nothing).
- One CCQ, one answer, move on (one learner ≠ everyone).
- Asking eliciting questions and calling them CCQs (different jobs — Module 2).
- "Do you understand?" / "OK?" as a check (unreliable).
- Teacher echoing every learner answer (lowers STT, trains learners to wait for the echo).
- Teacher answering their own question after 1 second of silence (give wait time).
- Concept-checking with the target grammar inside the question ("Is it happenING now?").
```

---

## OUTPUT STRUCTURE (for a given language point)
```
1. ELICITATION ROUTE: the context/visual + the prompts to draw the item out, with a fallback
   (narrow → supply → re-check) if elicitation fails.
2. CCQ SET: 3–4 CCQs, target-language-free, each with the expected answer; tested against a
   distractor meaning.
3. ICQ SET: for the practice task (form? pairs? time? output?).
4. DRILLING NOTES: model, stress mark-up, connected-speech/backchaining points.
```

## FAILURE MODES
- Conflating eliciting / CCQ / ICQ → keep the three jobs separate (Module 2).
- CCQ with the target language inside it → rewrite in plainer words (Module 4).
- Checking meaning but never drilling pronunciation → add Module 6.
- Killing momentum on a failed elicitation → narrow once, then supply + re-check (Module 3).
- No wait time → learners stop trying; the teacher does the work.
