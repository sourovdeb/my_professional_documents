## 🎯 Guidelines

1. **Active Voice**: "I discovered X" not "It was found that X"
2. **Human Tone**: Conversational, personal, not technical jargon
3. **Research-Based**: All claims from official/peer-reviewed sources
4. **Personalized**: Mention your bipolar/depression context where relevant
5. **Token-Efficient**: Clear, concise, no fluff
6. **500 Words**: Flexible ±50 words

---

## 📋 Essay Structure (TEMPLATE)

```
---
title: [Essay Title]
date: YYYY-MM-DD
category: [Mental Health / Automation / Philosophy / Wellbeing / Career]
tags: [tag1, tag2, tag3]
status: draft | published
wordpress_id: [if published]
---

# [Essay Title]

**Subtitle/Hook** (1-2 sentences establishing the idea)

## Opening (2-3 sentences)
Start with a personal observation or question relevant to your life.

## Body (3-4 paragraphs)
- Paragraph 1: Main idea + context
- Paragraph 2: How this applies to bipolar/mental health management
- Paragraph 3: Practical actions or tools
- Paragraph 4 (optional): Broader implications

## Closing (2-3 sentences)
Actionable takeaway. How will the reader apply this today?

---

**Word Count**: XXX | **Read Time**: X min | **Sources**: [links to official sources]
```

## 🔄 Publishing Workflow

1. Write in `2026/06/2026-06-03-topic.md` (this branch)
2. Update Google Drive sheet: add row with metadata
3. Review your essay (is it active voice? human tone?)
4. Commit: `git add Essays_and_Blogs/2026/06/2026-06-03-topic.md`
5. Push to feature branch
6. Copy .md to WordPress Drafts/ folder
7. (You copy to WordPress manually or via automation)
